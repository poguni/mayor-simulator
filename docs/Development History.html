1. 프로젝트 개요 및 교육적 목표

본 시뮬레이터는 초등 사회과 지역 사회의 발전 및 국토의 균형 발전 교육과정과 밀접히 연계하여 설계된 웹 게임 어플리케이션입니다.

학생들은 각각 대도시, 중간도시, 소도시의 시장으로 취임하여 5개년 임기 동안 시 예산을 집행하고, 주민의 복지(행복도)를 조율하며, 돌발 위기를 극복하는 과정을 통해 공공 재정의 유한성과 각 의사결정에 따르는 기회비용(Trade-off)을 직관적으로 학습합니다.

주요 핵심 가치: 단순히 돈을 아끼는 짠돌이 행정이나 과밀화를 조장하는 개발 중심 행정에 대해 세부 채점 알고리즘에서 감점을 주어, 상생과 복지를 균형 있게 고려하는 다방면의 종합 행정력을 스스로 체득하게 만듭니다.

2. 기술 스택 및 시스템 아키텍처

학교 현장의 크롬북(디벗) 및 태블릿 환경에서의 접근 편의성을 위해 별도의 복잡한 설치가 전혀 필요 없는 정적 페이지 구동 모델을 사용합니다.

  • 프런트엔드: 순수 HTML5, Vanilla CSS3, Vanilla JavaScript (ES6)
  • 데이터 시각화: Chart.js (최종 엔딩 보고서에서 인구, 예산, 행복도의 5개년 변화 선 그래프 생성)
  • 백엔드 (Serverless): Google Sheets (DB 역할) & Google Apps Script (GAS) (API 인터페이스 및 삭제 컨트롤러)

서버 가동 시 로컬 웹 서버(`http-server`) 환경을 권장하는 이유는, 로컬 파일 모드(file:///)로 열었을 때 브라우저의 보안 정책으로 인해 발생할 수 있는 Chart.js 라이브러리 및 NanumSquare 폰트의 로드 차단을 우회하기 위함입니다.

3. 프런트엔드 UI/UX 개편 이력

크롬북과 태블릿 해상도를 적극적으로 수용하여 레이아웃이 깨지거나 복잡해 보이지 않도록 전면 개편했습니다.

① 3단 게임 화면 및 좌측 현황판 개편

  • 화면 구성: 좌측(실시간 시정 지표 보드), 중앙(달 결재 카드 리스트), 우측(비서실 기안 서류 및 서명 영역) 배치.
  • 좌측 현황판 스크롤바 제거: 세로 높이가 초과하여 휠을 스크롤 해야 하던 문제를 해결하기 위해, 아바타 크기를 축소(56px ➡️ 44px)하고, 각 지표 박스 패딩을 줄이고, 숫자 폰트 크기(`1.7rem` ➡️ `1.45rem`)를 축소하여 스크롤바 없이 한눈에 들어오도록 압축 완료했습니다.

② 최종 시정 결과 보고서 4단 분할

  • 1단 (요약 & 지표): 왼쪽 패널에는 종합 등급 및 분야별 100점 점수를, 오른쪽 패널에는 시작 대비 최종 지표 증감량(예: 시작 대비 +4.5만 명, 시작 대비 -15억 원)을 대칭적으로 표기하여 품격을 높였습니다.
  • 2단 (그래프 횡정렬): 세로로 3줄 늘어져 스크롤을 유발하던 Chart.js 그래프 3개를 가로 3x1 레이아웃으로 나란히 배치했습니다.
  • 3단 (와이드 타임라인): 5년간의 전체 결재 행적을 가로 전체 폭을 활용한 와이드형 타임라인으로 보여주어 포트폴리오 효과를 극대화했습니다.
  • 4단 (생각 정리 에세이): 질문 아래에 입력창이 바로 정렬되도록 수정하고, 텍스트 입력 칸에 Pretendard 폰트를 강제 매핑했습니다. 제목 또한 [스스로 활동] 나의 생각을 정리해 봅시다.로 변경했습니다.

③ A4 가로 인쇄(Print) 기본 레이아웃 지정

프린트 쿼리(`@media print`)에 @page { size: landscape; margin: 10mm; } 속성을 적용하여, '보고서 인쇄하기' 클릭 시 기본 인쇄 용지 방향이 가로(Landscape)로 자동 고정되도록 디자인 시스템을 다듬었습니다.

4. 시뮬레이션 로직 및 밸런스 튜닝 이력

게임성 향상과 좋은 결정에만 점수가 지나치게 쏠리는 현상을 방지하기 위해 정밀한 밸런싱을 거쳤습니다.

① 중복 정책 추천 차단 및 풀 확장

임기 중 결재를 마친 정책 카드는 executedPolicyIds 배열에 등록하여 차기 연도 드로우에서 중복 추천되지 않도록 완전히 배제했습니다. 이를 뒷받침하기 위해 정책 카드 리스트를 총 18종으로 확장 수립했습니다. (재정 절약 대처 카드는 제외)

② 세입 인플레이션 억제

무조건 예산 관리 100점이 도출되던 예산 쏠림 현상을 예방하기 위해, 매년 추가되는 정기 세금을 다음과 같이 현실적으로 낮췄습니다.

  • 대도시: 매년 90억 ➡️ **50억 원**
  • 중간도시: 매년 45억 ➡️ **30억 원**
  • 소도시: 매년 18억 ➡️ **10억 원**

③ 돌발 위기 대처 방치 시 행복도 페널티 대폭 상향

예산을 지출하지 않고 방송 권고 등으로 때우는 대안을 선택할 경우 시민 행복도가 크게 삭감되도록 하여 행정적 기회비용을 실감하도록 튜닝했습니다.

특히 소도시 침수 피해(`e2`)를 방치했을 때 행복도가 올라가던 오류를 바로잡아, **행복도가 15점 감소**하고 원망 섞인 현장 스토리가 출력되도록 수정했습니다.

④ 지방 재정 '짠돌이 경영' 감점 및 마이너스 적자 허용

돈을 아예 안 쓰고 모으기만 하는 경우 공공 투자를 태만히 한 것으로 간주하여 예산 세부 점수를 최대 80점으로 제한하고, 예산 고갈 시 게임이 영구 멈추는 데드락을 방지하고자 마이너스 재정 적자 상태 진입을 허용하는 현장 해결 로직을 탑재했습니다.

5. 구글 스프레드시트(GAS) 백엔드 및 관리자 기능 연동 이력

학생들의 소중한 의사결정 포트폴리오를 웹 서버 없이도 교사가 통제하고 영구 보존할 수 있도록 구글 클라우드를 백엔드로 결합시켰습니다.

① 실시간 누적 저장 및 데이터 압축

학생이 최종 제출 시 API를 통해 구글 시트로 데이터를 전송할 때, 5년 임기 지표 추이(`historyJson`)와 기안 선택 타임라인(`decisionsJson`)을 JSON으로 압축하여 시트 후반 열에 보관하는 설계를 반영했습니다.

② 교사용 관리자 로그인 스크린 신설

메인 화면 우측 하단 링크 진입 시 전용 로그인 화면(`screen-admin-login`)이 뜹니다.
아이디: admin | 비밀번호: admin1204 계정 인증 모듈을 탑재하고, Enter 키 작동 편의성을 접목했습니다.

③ 학생 보고서 완벽 복원 및 개별 삭제 시스템

  • 보고서 조회 복원: 관리자가 리스트에서 특정 학생의 [조회]를 누르면, `gameState`에 그 학생의 역사 JSON과 수치들이 주입되어 엔딩 리포트 화면이 완벽하게 재현되며, [대시보드로 돌아가기] 버튼을 통해 현황판 복귀가 자유롭습니다.
  • 행 삭제 구현: 리스트에 빨간색 [삭제 🗑️] 버튼을 추가했습니다. 클릭 시 구글 스프레드시트에 고유 타임스탬프를 매칭시켜 **해당 학생 데이터 행을 구글 시트에서 직접 제거**하고 목록을 실시간 리프레시합니다.

6. CORS 프리플라이트 우회 및 디버깅 역사

연동 과정에서 구글 Apps Script 특성으로 인해 발생하는 보안 정책 오류들을 다년간의 실무 테크닉을 활용해 극복했습니다.

① CORS Preflight(OPTIONS 요청) 차단 회피

브라우저 fetch 통신 시 헤더에 JSON 전송 타입을 명시할 경우, 브라우저가 Preflight OPTIONS를 유발시켜 통신이 차단되는 치명적 한계가 존재했습니다. 이를 해결하고자 전송 Type을 브라우저 사전 심사를 받지 않는 단순 요청 규격인 **`text/plain`**으로 매칭하고 **`no-cors`** 전송 모드를 적용하여 보안 차단을 원천 우회했습니다.

② Code.gs 실행 오류 제거

구글 스크립트 엔진 내에서 지원되지 않는 .setHeader() 메소드 호출 시 발생하던 TypeError(500 에러)를 제거했습니다. GAS의 TextOutput 반환 방식은 자동으로 CORS가 허용되므로, 에러 유발 지점을 지워 완벽한 호출 성공을 달성했습니다.

③ 백그라운드 Null 인스턴스 에러 극복

구글 서버가 비동기로 스프레드시트 접근을 시도할 때 활성 인스턴스를 찾지 못하던(getActiveSpreadsheet ➡️ null) 문제를 방지하기 위해, 스프레드시트 고유 ID 명시적 바인딩 변수를 코드 상단에 신설하고 활성 시트 대신 물리적으로 첫 번째 탭 시트만 강제 지목하는 **`getSheets()[0]`** 연산자를 매핑했습니다.
