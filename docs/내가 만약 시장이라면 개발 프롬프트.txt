[프롬프트 #01]

[프롬프트 #02]
  너는 유능한 10년차 시뮬레이션 게임 개발 전문가야. 초등학교 고학년 학생들이 도시의 시장을 맡아 다양한 정책을 선택하면서 도시의 균형 발전을 시뮬레이션 해 볼 수 있는 학습용 앱을 개발하려고 해. Plan.md 파일과 Analysis Results.md의 내용을 참고해서 먼저 화면 구성을 해 줘. 데이터는 지금 다 만들지 말고 더미 데이터로 넣어도 괜찮아.

[프롬프트 #03]
  너는 유능한 도시 개발 기획자이자 15년 경력의 시청 공무원이야. 새로 부임한 시장이 다양한 분야에서 정책을 추진할 수 있도록 시뮬레이션 내용을 풍부하게 만들어 줘. 정책과 효과를 만들 때에는 어느 정도 현실성 있게 알맞은 예산을 배정하고, 정책이 실현되었을 때 시민들의 만족도도 알맞게 반영해 줘. 
  이 부분은 시뮬레이션 앱의 핵심이야. 학생들이 시장으로서 도시 경영을 시뮬레이션 해볼 수 있도록 풍부한 자료를 만들어 반영해 줘.

----------

[프롬프트 #04]
  너는 교직 경력 15년차의 베테랑 초등학교 선생님이야. 이 앱의 내용을 세심하게 살펴 보고 초등학교 고학년 학생들에게 내용이 적합한지 확인해 줘. 초등학생 수준에 알맞게 어려운 용어는 쉽게 바꾸어 주고, 너무 전문적인 내용은 포함하지 않도록 주의해 줘. 또, 초등학생에게 알맞게 글의 길이도 적절히 조절해 줘.

----------

[프롬프트 #05]
  다시 시뮬레이션 앱 개발 전문가에게 작업이 넘어 왔어. 앞에서 도시 개발 기획자와 베테랑 선생님이 내용을 확인했어. 너는 이 내용이 앱에 잘 반영될 수 있도록 다시 한 번 확인해 줘.
  - 화면의 글씨를 조금씩 크게 해 줘. 디벗의 특성 상 작은 글씨가 잘 보이지 않으니 글씨를 좀 더 키워줘.
  - 정책을 승인하고 나오는 신문 기사의 제목을 '[도시이름] 신문'으로 바꾸어 줘. 그리고 'APPROVED' 대신에 '승인'으로 바꾸어 줘.
  - 초기 화면의 배경 화면을 다음 파일로 바꾸어 줘.
    MayorSimulator\src\assets\title.jpg
  - 메인 화면에서 시장의 이름을 더 크게 해 줘. 아이들이 자부심을 가질 것 같아. 
  - 시정 결과 최종 보고서를 인쇄할 때 A4 용지 2~3쪽에 나올 수 있도록 비율을 조정해 줘.
  - 맨 마지막 화면에서 '시정 마무리 생각 정리 (사회과 평가 질문)'의 제목과 질문, 설명의 글씨 크기를 모두 크게 해 줘. 매우 중요한 평가 질문인데 디벗 화면에서 잘 보이지 않을 것 같아. 학생들이 입력하는 텍스트 상자에도 글씨가 크게 들어갈 수 있도록 수정해 줘.

[프롬프트 #06]
  다시 도시 개발 기획자에게 부탁할게. 시뮬레이션 앱의 특성 상 어느 정도 현실성이 반영되어야 할 것 같아.
  - 동일한 정책이 계속 등장하고 있어. (예. 밤에도 여는 어린이 병원 세우기를 1년차에 선택했는데 다음 해에도 계속 등장하고 있어. 교육적인 내용을 담아야 하기 때문에 한계가 있겠지만 그래도 다양한 정책을 만들어서 추가해 줘.)
  - 5년 동안 예산을 계속 집행했는데 5년 후에 남은 예산이 더 많아지는 경우가 있어. 매년 자동으로 추가되는 금액을 줄이고, 도시 규모별로 예산 소모액을 적절히 수정해 줘.
  - 초등학생이라 좋은 평가를 주는 것은 괜찮은데 거의 다 100점이 나와. 너무 낮은 점수도 문제지만 무엇을 선택하던 100점 가까이 나오는 것도 현실성이 없는 것 같아. 나의 시장 등급 점수를 현실적으로 바꾸어 줘.

[프롬프트 #07]
  너는 다년간 수많은 프로젝트를 성공적으로 수행한 백엔드 전문가야.   
  1. 이제 이 앱에서 발생한 데이터를 저장하려고 해. 먼저 
    - 시장의 이름과 도시 이름
    - 시장이 선택한 정책들
    - 시정 결과 최종 보고서의 내용
    - '[스스로 활동] 나의 생각을 정리해 봅시다.'에서 학생들이 질문에 응답한 내용
    나중에 교사용 관리자 페이지에서 학생들의 결과 최종 보고서를 다시 인쇄/저장할 수 있도록 모든 데이터를 저장해 줘야 해.

  2. 데이터 저장은 Plan.md에서 나온 대로 Google Apps Script를 사용할 거야. 
    - 데이터 저장 (학생 → Sheets) : 학생이 제출을 완료하면, 브라우저가 GAS 웹 앱 URL로 POST 요청을 보내 결과를 Google Sheet의 새 행으로 추가할 거야.
    - 데이터 조회 (Sheets → 교사 대시보드) : 교사가 관리자 로그인 후 대시보드에 접근하면, app.js가 GAS 웹 앱 URL로 GET 요청을 보내 모든 학생의 결과 데이터를 실시간으로 가져와 화면을 그리면 돼.

  3. Google Apps Script에 사용할 Code.gs 스크립트를 작성해 줘.
  4. 생성된 웹 앱 URL을 어디에 복사해 넣어야 하는지도 알려줘.
