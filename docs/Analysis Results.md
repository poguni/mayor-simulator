# 📊 '내가 만약 시장이라면' 앱 기획 분석 보고서

본 보고서는 `docs/Plan.md` 파일에 정의된 초등 사회 교과 연계형 교육 시뮬레이션 앱 **'내가 만약 시장이라면'**의 특징과 기능을 분석하여 정리한 문서입니다.

---

## 📌 1. 프로젝트 개요
* **프로젝트명:** 내가 만약 시장이라면 (부제: 우리 도시 경영 시뮬레이터)
* **타깃 사용자:** 초등학교 4~6학년 학생 및 지도 교사
* **교육적 목적:** 초등 사회 교과(지방 자치, 도시 발달과 문제, 국토 균형 발전) 연계 학습. 제한된 예산 속에서 정책을 선택하고 도시 변화를 경영해보며 '국토 균형 발전의 필요성'과 '합리적 의사결정 능력'을 자기주도적으로 학습.
* **실행 환경:** 디벗(크롬북 등) FHD (1920x1080) 해상도 최적화 웹 애플리케이션 (HTML5/CSS3/JavaScript 기반)

---

## 🎨 2. 디자인 및 시각적 특징
* **컨셉:** 친근한 행정 집무실 (**Soft Bureaucracy**) – 플랫하고 둥근 형태의 부드러운 UI
* **폰트:** 타이틀/주요 알림은 친근한 `Nanum Square Round`, 본문/수치는 가독성이 높은 `Pretendard` 적용
* **컬러 시스템:**
  * **Main (신뢰):** Soft Blue (`#4A90E2`)
  * **Positive (성장/상승):** Mint Green (`#2ECC71`)
  * **Negative (경고/감소):** Soft Coral (`#E74C3C`)
  * **Background:** 연한 회색 (`#F8F9FA`) 기반의 카드 UI 배치
* **비주얼 연출:** 화려한 그래픽보다는 텍스트와 직관적인 이모지(Emoji)를 결합하여 초등학생의 눈높이에 최적화

## ⚙️ 3. 핵심 시뮬레이션 메커니즘

### ① 도시 규모별 초기 설정 및 교과 연계 패널티
사용자가 선택한 도시 규모에 따라 시작 데이터와 고유의 사회적 문제가 부여됩니다.

| 도시 규모 | 초기 인구 | 초기 예산 | 인구 밀도 | 고유 패널티 및 특징 (교과 연계) |
| :--- | :---: | :---: | :---: | :--- |
| **대도시** | 50만 명 | 1,000억 원 | 🔴 복잡함 | 과밀화 문제 (매년 교통·환경 수치 자동 감소) |
| **중간 도시** | 30만 명 | 500억 원 | 🟡 적당함 | 인구 유출 위험 (대도시로의 이동 경계 필요) |
| **소도시** | 10만 명 | 100억 원 | 🟢 한적함 | 지방 소멸 위기 (일자리·교육·의료 등 기본 인프라 부족) |

### ② 정책 카드 트레이드오프 구조
제시되는 모든 정책은 이익과 손실이 공존하여 합리적 의사결정을 유도합니다. (텍스트는 최대 3줄 이내)
* **공공 기관 이전/유치:** 대도시는 인구 분산/환경 개선, 소도시는 인구 유입/행복도 상승 유도
* **로컬 브랜딩 사업 지원:** 지역 문화 자원 활용. 예산 대비 주민 행복도와 도시 활력 대폭 상승
* **고향 사랑 기부제 시행:** 대도시는 효과 미미하나 소도시는 매년 지속적인 세입(예산) 보너스 제공
* **청년 마을 만들기:** 소도시에 필수적인 인구 유출 방지 및 외부 청년 유입 정책
* **교육/편의/문화 시설 확보:** 학교·병원·도서관 건립. 미선택 시 매년 주민 행복도가 지속 감소

### ③ 5개년 진행 루틴
* 매년 **총 3건**의 의사결정(정책 2회 + 돌발 이벤트 1회)을 수행
* **진행 흐름:** `[올해의 정책 선택]` ➡️ `[선택 결과 보고서 확인]` ➡️ `[깜짝 돌발 이벤트 해결]` ➡️ `[두 번째 정책 선택]` ➡️ `[결과 보고서 확인]` ➡️ `[다음 년차 이동]`

## 🖥️ 4. 주요 화면 구성 및 기능

### 화면 1: 로그인 (초기 화면)
* 시장 이름 및 도시 이름 입력
* **[🎲 랜덤 추천]** 버튼 제공 (순우리말 도시 이름 리스트 매칭을 통해 초등학생의 작명 고민 해결)

### 화면 2: 도시 규모 선택 화면
* 대도시 / 중간 도시 / 소도시를 설명하는 3개의 카드 배치
* 각 카드는 특징(예: 소멸 위험 등)과 초기 데이터를 시각화하여 제시
* 하단에 `[OO 도시의 시장으로 취임하기 👑]` 버튼 제공

### 화면 3: 메인 게임 화면 (★ 3단 레이아웃 분할)
FHD 가로 화면을 2:5:3 비율로 분할하여 복잡도를 낮추고 정보 전달력을 높임.
* **좌측 패널 (20% - 우리 도시 현황판):** 시장/도시 이름, 임기(예: 3년 차), 핵심 지표(예산 💰, 인구 👥, 인구 밀도 🚗 - 신호등 색상 적용, 행복도 🥰)를 상시 표시
* **중앙 패널 (50% - 집무실 정책 제시):** 5가지 정책 카드 목록 제공 및 클릭 시 활성화
* **우측 패널 (30% - 비서실 결재 서류):** 선택한 정책의 상세 설명(줄글 2줄)과 예상 변경 수치(예: 예산 -15억 / 행복도 +10%)를 보여주고, 하단에 `[정책 실행 결재하기 🖋️]` 버튼 제공

### 화면 4: 정책 시행 결과 보고서 (결과 확인 팝업)
* 정책 결재 시 도장 소리와 효과 연출 후 전체 화면 팝업 등장
* **가상 신문 헤드라인 기사** 형태로 재미있게 연출 (예: *"창업 청년들 몰려온다! OOO 시장의 청년마을 대성공"*)
* 실제 변경된 수치를 직관적인 화살표(`📈`, `📉`)와 함께 표기

## 📜 5. 엔딩 및 사후 학습 기능

### ① 최종 평가 시스템 (5년 임기 종료)
* 누적 데이터를 바탕으로 **최종 등급(타이틀)**과 **분야별 별점(5성 만점)** 부여
  * *타이틀 예시:* `🏆 황금 밸런스 명품 시장`, `🍀 초록 환경 전문가 시장`, `🔥 황금빛 경제 활력 시장` 등
* **시각화 지표:** 5년간의 인구, 행복도, 예산 변화 추이를 직관적으로 확인할 수 있는 선 그래프(Line Chart) 3개 나열
* **역사 기록:** 5년간 선택했던 정책들의 연도별 타임라인 표시

### ② 교과 마무리 질문 (서술형 답변)
보고서 최하단에 학생들이 생각을 정리하고 느낀 점을 적는 3가지 질문 탑재:
1. 국가의 균형 발전을 위해 가장 중요한 정책이 무엇이라고 생각하나요?
2. 왜 그렇게 생각하나요?
3. 5년 동안 시장을 맡아 시를 운영하고 느낀 점을 적어봅시다.

## 🛠️ 6. 시스템 아키텍처 및 기술적 요구사항

### ① 백엔드 대체: Google Apps Script (GAS) 연동
* **구동 방식:** 학습 완료 후 `[선생님께 제출하기 📤]` 클릭 ➡️ `fetch()` API로 GAS에 데이터 전송 ➡️ 교사의 구글 스프레드시트에 행(Row) 단위로 자동 기록
* **저장 데이터:** 타임스탬프, 시장 이름, 도시 유형, 최종 등급, 최종 인구수, 최종 행복도, 최종 잔여 예산, 서술형 답변 1~3
* **안정성:** 동시 제출로 인한 데이터 누락을 방지하기 위해 GAS에 `LockService` 구현 필수

### ② 인쇄 기능 최적화 (Print CSS)
* `@media print` 전용 스타일 시트를 설계하여 결과 보고서 출력 지원 (오프라인 포트폴리오로 활용)
* 제출/이동 등 UI 버튼 숨김 (`display: none;`)
* 배경 파스텔 톤 제거 및 텍스트 검은색 전환으로 흑백 프린터 대응
* A4 용지 범위 내에서 그래프나 타임라인이 잘리지 않도록 `page-break-inside: avoid;` 정의
