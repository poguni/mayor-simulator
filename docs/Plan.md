# 📑 [개발 기획서] 내가 만약 시장이라면 (우리 도시 경영 시뮬레이터)

## 📌 1. 프로젝트 개요

* **프로젝트명:** 내가 만약 시장이라면 (부제: 우리 도시 경영 시뮬레이터)

* **개발 목적:** 초등 사회 교과(지방 자치, 도시 발달과 문제, 국토 균형 발전)와 연계하여, 학생들이 대·중·소도시의 시장이 되어 제한된 예산 안에서 정책을 선택하고 도시의 변화를 직접 시뮬레이션함으로써 균형 발전의 필요성을 자기주도적으로 학습하도록 함.
* **실행 환경:** 학생들이 사용하는 디벗(크롬북 등)의 FHD (1920x1080) 해상도 최적화 웹 애플리케이션 (HTML5/CSS3/JavaScript 기반)

## 🎨 2. 디자인 톤 앤 매너 (Tone & Manner)

* **글꼴(Typography):**
* `Nanum Square Round`: 타이틀, 알림 문구, 주요 버튼 (친근하고 귀여운 느낌 강조)
* `Pretendard`: 본문 설명, 지표 수치 (가독성과 정보 전달력 최적화)


* **컬러 팔레트:**
* **Main (신뢰):** Soft Blue (`#4A90E2`) - 전체적인 시스템 및 상단 바
* **Positive (성장/상승):** Mint Green (`#2ECC71`) - 예산·인구·행복도 상승 표시
* **Negative (위기/경고):** Soft Coral (`#E74C3C`) - 예산 부족, 인구 감소, 재난 상황 표시
* **Background (눈 피로 방지):** 연한 회색 (`#F8F9FA`) 바탕에 흰색 카드 UI 배치


* **시각 요소:** 화려한 그래픽을 배제하는 대신 텍스트 옆에 직관적인 이모지(Emoji)를 결합하여 초등학생 눈높이에 맞춘 아기자기한 화면 구성.

## 🔀 3. 핵심 게임 메커니즘 및 데이터 구조

### ① 도시 규모별 초기 설정 데이터

| 도시 규모 | 초기 인구 | 초기 예산 | 인구 밀도 | 도시별 고유 패널티 및 특징 (교과 연계) |
| --- | --- | --- | --- | --- |
| **대도시** | 50만 명 | 1,000억 원 | 🔴 복잡함 | 과밀화 문제 보유 (매년 교통·환경 수치 자동 감소 페널티) |
| **중간 도시** | 30만 명 | 500억 원 | 🟡 적당함 | 인구 유출 위험 보유 (대도시로의 인구 이동 경계 필요) |
| **소도시** | 10만 명 | 100억 원 | 🟢 한적함 | 지방 소멸 위기 보유 (일자리, 교육, 의료 인프라 부족 상태) |

### ② 정책 카드 데이터 풀 (매년 5개 중 1개 최종 결재)

모든 정책은 트레이드오프(하나를 얻으면 하나를 잃는 구조)를 가집니다. *텍스트는 최대 3줄 이내로 제한.*

* **공공 기관 이전/유치 (비용 10억~30억):** 대도시는 인구 분산 및 환경 개선효과 / 소도시는 대규모 인구 유입 및 행복도 상승 효과.
* **로컬 브랜딩 사업 지원 (비용 5억):** 지역 문화 자원 활용. 예산 소모 대비 주민 행복도와 활력도 대폭 상승.
* **고향 사랑 기부제 시행 (비용 3억):** 대도시에는 효과 미비 / 소도시는 다음 해부터 매년 지속적인 세입(예산) 보너스 발생.
* **청년 마을 만들기 (비용 5억):** 지역 청년 유출 방지 및 외부 청년 유입. 소도시에 필수적인 인구 방어 정책.
* **교육/편의/문화 시설 확보 (비용 15억~20억):** 학교, 종합병원, 도서관 건립. 미선택 시 매년 주민 행복도가 감소하는 기본 인프라 정책.

### ③ 5개년 진행 루틴 (매년 총 3건의 의사결정)

* **매년 진행 순서:** `[올해의 정책 선택]` ➡️ `[선택 결과 보고서 확인]` ➡️ `[깜짝 돌발 이벤트 등장 및 해결]` ➡️ `[두 번째 정책 선택]` ➡️ `[결과 보고서 확인]` ➡️ `[다음 년차 이동]`

## 🖥️ 4. 화면별 상세 기획 (Wireframe 및 UX)

### 화면 1: 초기 화면 (로그인)

* **구성:** 화면 중앙에 깔끔한 로그인 카드 배치.
* **기능:** `시장님 이름`과 `도시 이름` 입력창 제공.
* *초등 눈높이 기능:* 도시 이름을 고민하는 학생들을 위해 **[🎲 랜덤 추천]** 버튼 제공 (예: 미리내시, 푸른샘시 등 순우리말 단어 리스트 매칭).

### 화면 2: 도시 규모 선택 화면

* **구성:** 가로로 대도시 / 중간 도시 / 소도시를 나타내는 3개의 큰 카드 배치.
* **내용:** 각 카드 내부에는 도시별 특징(예: "소도시는 자연이 아름답지만 소멸 위험이 있어요!")과 초기 데이터(인구, 예산, 밀도)가 보기 쉽게 정렬됨.
* **액션:** 카드 하단에 나눔스퀘어 라운드 서체의 `[OO 도시의 시장으로 취임하기 👑]` 버튼 배치.

### 화면 3: 메인 게임 화면 (★핵심 3단 레이아웃)

FHD 가로 화면을 2:5:3 비율로 정밀 분할하여 정보 복잡도를 낮춤.

* **왼쪽 패널 (비율 20% - 우리 도시 현황판):**
* 시장 이름, 도시 이름, 현재 임기(예: 3년 차) 표시.
* 💰 예산, 👥 인구, 🚗 인구 밀도(신호등 색상 적용), 🥰 시민 행복도를 큰 글씨와 이모지로 고정 표시.


* **가운데 패널 (비율 50% - 집무실 정책 제시):**
* 안내 문구: *"OOO 시장님! 아래 5가지 정책 중 한 가지를 선택해 주세요..."*
* 세로 또는 격자형으로 배치된 5개의 정책 카드 리스트. 클릭 시 선택 상태 활성화.


* **오른쪽 패널 (비율 30% - 비서실 결재 서류):**
* 가운데 패널에서 카드를 클릭하면 해당 정책의 상세 내용 표출.
* `[줄글 설명 2줄]` + `[예상 변경 수치 지표]` (예: 예산 -15억 / 행복도 +10%)
* 하단에 눈에 띄는 색상의 **`[정책 실행 결재하기 🖋️]`** 버튼 배치.


### 화면 4: 정책 시행 결과 보고서 (팝업 연출)

* 결재 버튼 클릭 시 도장 소리와 함께 2~3초간 애니메이션 진행 후 전체 화면 팝업 등장.
* **콘텐츠:** 가상의 신문 헤드라인 기사 형태로 연출하여 흥미 유발 (예: *"창업 청년들 몰려온다! OOO 시장의 청년마을 대성공"*). 실제 변동된 수치를 직관적인 화살표(`📈`, `📉`)와 함께 표기.

## 🛠️ 6. 시스템 아키텍처 및 기술적 요구사항

### ① 백엔드 대체: Google Apps Script (GAS) 연동

데이터베이스 구축 비용을 절감하고 교사의 관리 편의성을 극대화하기 위해 구글 스프레드시트를 백엔드로 활용합니다.

* **작동 프로세스:** 학생이 시뮬레이션을 마치고 결과 화면에서 `[선생님께 제출하기 📤]` 버튼을 누르면, JavaScript의 `fetch()` API를 통해 GAS로 데이터 전송 ➡️ 교사의 구글 스프레드시트에 행(Row) 단위로 실시간 저장.
* **저장 데이터 스키마:** `타임스탬프`, `시장 이름`, `도시 유형`, `최종 등급`, `최종 인구수`, `최종 행복도`, `최종 잔여 예산`, `서술형 답변 1`, `서술형 답변 2`, `서술형 답변 3`
* **안정성 확보:** 동시 다발적인 학생들의 제출 요청으로 인한 데이터 누락을 방지하기 위해 GAS 코드 내에 `LockService`를 반드시 구현함.

### ② 프린트(Print) 기능 사양 (Print CSS)

결과 보고서를 출력하여 오프라인 포트폴리오로 활용할 수 있도록 브라우저 인쇄 기능을 최적화합니다.

* **`@media print` 전용 스타일 시트 적용:**
* 화면상의 '목록으로', '제출하기' 등 조작용 버튼 요소 일체 숨김 처리 (`display: none;`).
* 학교 인쇄 환경(흑백 프린터)을 고려하여 모든 배경 파스텔 색상을 제거하고 텍스트를 고대비 검은색으로 강제 전환.
* A4 용지 규격에 맞춰 그래프와 타임라인이 페이지 경계선에서 애매하게 잘리지 않도록 `page-break-inside: avoid;` 속성 정의.
