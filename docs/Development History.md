## 1. 프로젝트 개요 및 교육적 목표

본 시뮬레이터는 초등학생들이 직접 가상의 도시(대도시, 중간도시, 소도시)의 시장으로 취임하여 5년 임기 동안 다양한 정책 결재와 돌발 위기 대처를 해결하며, **도시 규모별 사회 문제와 공공 예산 분배의 기회비용**을 몸소 이해할 수 있도록 설계된 교육용 웹 애플리케이션입니다.

* **교육적 연계:** 초등 사회과 지역 사회의 발전 및 기후 변화, 지방 소멸, 국토의 균형 발전 단원.
* **핵심 학습 가치:** 
  1. 공공 예산(지방 재정)은 유한하며 모든 투자는 트레이드오프(기회비용)가 발생한다는 점을 이해.
  2. 주민 복지(행복도) 향상과 무분별한 개발 사이의 균형감 학습.
  3. 시정 종료 후 서술형 작성을 통한 메타인지 및 자기성찰 포트폴리오화.

## 2. 기술 스택 및 시스템 아키텍처

애플리케이션의 설치 편의성(교실 내 크롬북 등 단독 구동 환경)을 극대화하기 위해 별도의 무거운 WAS 서버 설치 없이 **완전한 무설치 정적 클라이언트 + 서버리스(Serverless) 구동 아키텍처**를 수립했습니다.

* **Client Side:** 
  * `HTML5` (구조화 및 접근성 보장)
  * `Vanilla CSS3` (프리미엄 HSL 기반 다크/라이트 하모니 스타일링, 반응형 그리드)
  * `Vanilla JavaScript (ES6)` (게임 상태 제어 및 인터랙션)
  * `Chart.js` (최종 보고서 내 5개년 지표 변화 선 그래프 시각화 라이브러리)
* **Back End (Serverless):**
  * `Google Sheets` (학생들의 데이터 영구 적재용 데이터베이스 역할)
  * `Google Apps Script (GAS)` (스프레드시트 API 인터페이스 역할 - doGet, doPost 엔드포인트 지원)

## 3. 프런트엔드 UI/UX 개편 이력

디바이스 해상도가 다양한 교육 현장(특히 크롬북, 태블릿, 디벗 환경)의 특성을 반영하여 레이아웃을 고도화했습니다.

### ① 메인 게임 화면 3단 패널 그리드 최적화
* **좌측 패널 (실시간 시정 현황 - 20%):** 
  * 예산, 인구수, 인구 밀도, 행복도를 한눈에 트래킹하는 보드입니다.
  * *업데이트 사항:* 원래 세로로 지나치게 길어 스크롤바가 생기던 구조를 개선하기 위해 아바타 크기 축소(56px ➡️ 44px), 임기 배지 두께 감소, 각 지표 박스(`stat-box`)의 안쪽 여백(`padding`) 및 지표 값 폰트 크기(`1.7rem` ➡️ `1.45rem`)를 콤팩트하게 다듬어 **스크롤바 없이 한눈에 들어오도록 스케일을 조율**했습니다.
* **중앙 패널 (집무실 정책 후보 - 50%):** 매 연차 무작위 추천된 5개 카드 중 하나의 정책을 비교 분석하여 선택하는 코어 구역입니다.
* **우측 패널 (비서실 결재 서류 - 30%):** 기안 서류 양식을 띄우고 서명 도장을 타격하는 연출 공간입니다.

### ② 최종 보고서(Screen 5) 4단 레이아웃 개편
시정 종료 시 표시되는 보고서의 레이아웃을 1단부터 4단까지 시원하게 분할하여 대폭 개선했습니다.
* **1단 (대칭적 요약 리포트):**
  * **좌측 패널:** 나의 최종 등급(칭호), 5성 환산 별점 등급, 학생 행보에 맞춘 한 줄 칭찬 코멘트, 3대 시정 분야별 100점 만점 세부 스코어 표시.
  * **우측 패널:** 임기 시작 대비 최종 시점의 지표(`인구`, `행복도`, `예산`) 수치 비교 및 `+`/`-` 성과 증감률을 직관적으로 바인딩.
* **2단 (지표 선 그래프):** Chart.js를 사용한 인구, 예산, 행복도 3개의 선형 그래프를 **3x1 가로 한 줄 정렬**로 강제 배치하여 데이터 시각화의 연속성 제공.
* **3단 (결재 이력 타임라인):** 가로 와이드 100% 폭의 타임라인 그리드를 적용하여 5년간 결재한 내용과 해결한 이벤트를 히스토리로 기록.
* **4단 (스스로 활동 에세이):** 
  * 질문과 입력창의 배치가 깨지던 현상을 바로잡아 **질문 텍스트 하단에 입력 텍스트 상자가 오도록 정렬**했습니다.
  * 크롬북 기본 입력 폰트 대신 미려한 **Pretendard 글꼴**을 내부 텍스트에 강제 적용했습니다.
  * 제목을 기존의 '시정 마무리 생각 정리'에서 **`[스스로 활동] 나의 생각을 정리해 봅시다.`**로 초등 교육 과정에 맞게 순화했습니다.

### ③ 인쇄(Print) 레이아웃 및 뷰 통일
* `@media print` 쿼리를 대폭 튜닝하고 최상단에 `@page { size: landscape; margin: 10mm; }` 규격을 명시했습니다.
* 이로 인해 인쇄 창 호출(Ctrl+P) 시 브라우저 기본 레이아웃이 **'가로(Landscape)'**로 고정되어 차트와 타임라인이 용지 여백을 꽉 채워 기품 있게 인쇄됩니다.

## 4. 시뮬레이션 로직 및 밸런스 튜닝 이력

초기 버전에서는 좋은 선택지만 고르면 무조건 100점이 나오고 예산이 기하급수적으로 증가하여 시뮬레이션의 현실감과 교육적 긴장감이 떨어졌었습니다. 이를 해결하기 위해 총 3차에 걸친 정교한 밸런싱 작업을 진행했습니다.

### ① 중복 결재 정책 배제 및 카드 풀 확장
* **중복 추천 제거:** 플레이어가 결재를 마치면 해당 정책 ID가 `gameState.executedPolicyIds` 배열에 등록되어 차년도 추천 목록에서 완전히 배제됩니다. (예산 고갈에 따른 데드락 방지용 0원 카드인 `emergency_policy`는 제외)
* **풀 확장:** 중복이 배제됨에 따라 5개년 간 다채로운 추천이 보장되도록 정책 풀을 기존 10종에서 **18종**으로 대폭 확장했습니다.

### ② 세수 인플레이션 차단 및 비용 튜닝
* **연말 세수 보너스 (`taxIncome`) 조정:**
  * 대도시: 매년 150억 ➡️ 90억 ➡️ **최종 50억 원**
  * 중간도시: 매년 80억 ➡️ 45억 ➡️ **최종 30억 원**
  * 소도시: 매년 35억 ➡️ 18억 ➡️ **최종 10억 원**
* **정책 실행 비용 인상:** 대규모 개발 사업의 소모 금액을 약 20% 인상하여, 무분별하게 큰 사업만 추진할 시 실제로 국고가 바닥나 **재정 적자(마이너스)**가 나도록 설계했습니다.

### ③ 돌발 상황 (Surprise Event) 기회비용 조율
* 돌발 위기 시 예산을 쓰지 않고 방송 홍보나 강제 강행 등으로 대충 때우는 옵션(2번)을 골랐을 때 발생하는 시민의 **행복도 감점 페널티를 대폭 하향 조정**하여 투자 회피에 대한 명확한 반대급부를 인지하게 했습니다.
* 특히 소도시 홍수 발생(`e2`) 상황에서 예산을 안 썼을 때 기존에 '이웃의 정'이라는 미명 하에 행복도가 오히려 상승(`+8`)하던 오류를 고쳐, 수해 복구 방치에 따른 서운함으로 **행복도가 15점 대폭 감소**하도록 수정했습니다.

### ④ 등급 및 채점 알고리즘 고도화
* **인구 점수:** 단순히 인구를 늘리는 것이 장땡이 아니라 대도시의 경우 과밀화(인구 정체 및 매연 패널티)를 해소했는가, 소도시의 경우 소멸 위기(인구 하한선 방어)를 막았는가에 따라 변별력 채점.
* **예산 점수:** 짠돌이 경영(재정을 전혀 투자하지 않고 아끼기만 함)을 방지하기 위해 잔여 재정이 과도하게 많을 경우 최대 80점으로 한계선을 제한하여 '올바른 지방 재정 투자'의 중요성을 깨닫게 유도. 적자 발생 시 30점 고정 패널티.

### ⑤ 신규 정책 카드 10종 추가 및 도시 규모별 노출 필터링 적용 (2026-06-06)
* **도시 규모별 전용 정책 추가:** 대도시용 3종, 중간도시용 3종, 소도시용 4종의 정책 카드 총 10종을 신설하여 정책 풀을 28종으로 대폭 확장했습니다.
  * *대도시용(3종):* 대학 공동 캠퍼스 구축, 도심 공장 외곽 이전 지원, 귀농·귀촌 체험 지원센터 운영.
  * *중간도시용(3종):* 지역 혁신 기업(스타트업) 타운 조성, 24시간 돌봄 공립 어린이집 및 아동 종합병원 건립, 광역 급행 버스(BRT) 개통.
  * *소도시용(4종):* 은퇴자 복합 웰니스 실버타운 조성, 시골 유학 및 가족 체류 센터 운영, 스마트 원격 진료소 및 이동식 버스 병원 도입, 크리에이터 및 디지털 노마드 공유 오피스 구축.
* **도시 규모별 노출 필터링 구현:** 각 정책 객체에 `allowedCities` 속성을 정의하여, `renderPolicyList()` 함수에서 현재 플레이어의 `cityType`에 해당하는 전용 정책 및 공용 정책(속성이 없는 경우)만 필터링하여 노출되도록 정책 렌더링 시스템을 고도화했습니다.

### ⑥ 예산 부족 대안 옵션 구현: 건너뛰기 및 은행 대출 (2026-06-06)
* **건너뛰기 정책 상시 추가:** 예산 부족에 따른 데드락을 원천적으로 막기 위해, 매 턴 6번째 정책 후보로 "🚫 아무 정책도 실행하지 않기 (건너뛰기)" 옵션을 고정 렌더링하도록 처리했습니다. 선택 시 예산 소모는 0원이지만, 인구가 대폭 줄어들고 시민 행복도가 10~15점 하락하는 페널티가 발생합니다.
* **은행 대출(마이너스 예산) 연동:** 예산보다 비싼 정책을 결재하려고 할 때 대출 확인 창(confirm)을 띄우고, 확인 시 마이너스 예산으로도 집행이 가능하게 수정했습니다.
* **대출 한도 제한:** 은행 대출은 무제한이 아니라, 현재 턴에 추천된 일반 정책들 중 최대 비용을 넘을 수 없도록 안전 장치를 마련했습니다. 한도를 넘어서면 경고 메시지와 함께 결재가 차단됩니다.
* **최종 적자 감점 보정:** 시정 종료 시 최종 재정이 마이너스(적자)일 경우, `evaluateScore()` 내에서 예산 점수를 0점으로 하고 인구/행복도도 추가 15점씩 감점 처리합니다. 또한 칭호를 `💸 빚더미에 앉은 적자 시장`으로 강제 격하시키고 1.0성의 별점을 부여하여 빚더미 시정에 대한 강력한 교육 피드백을 주도록 알고리즘을 변경했습니다.

### ⑦ 도시 규모별 돌발 상황 이벤트 10종 추가 및 셔플링 연동 (2026-06-06)
* **도시 규모별 돌발 상황 추가:** 대도시용 3종, 중간도시용 3종, 소도시용 4종의 돌발 상황 카드를 신설하여 이벤트 풀을 15종으로 확장했습니다.
  * *대도시용(3종):* 지옥철 및 교통 마비 사태 발생, 도심 노후 아파트 대단지 정전/단수, 쓰레기 매립지 포화 위기.
  * *중간도시용(3종):* 신도시 배후 도로 싱크홀 발생, 구도심 전통시장 대형 화재, 소아과 원정 진료 대란.
  * *소도시용(4종):* 극심한 가뭄으로 인한 농업용수 부족, 유일한 고등학교 폐교 위기, 폭우로 인한 산사태 마을 고립, 외지인 쓰레기 무단 투기 계곡 오염.
* **이벤트 무작위 셔플링 및 바인딩 구현:** 중복 없이 다채로운 위기 대처 시뮬레이션을 보장하기 위해, `initGame()` 호출 시 현재 `cityType`에 허용되는 돌발 이벤트를 필터링하고 무작위로 섞어 `gameState.activeEvents` 배열에 5종을 미리 바인딩했습니다. 이후 매년 `triggerSurpriseEvent()`에서 해당 5종을 순서대로 트리거하도록 로직을 개편했습니다.

## 5. 구글 스프레드시트(GAS) 백엔드 및 관리자 기능 연동 이력

학급의 담임 교사가 학생들의 수행 결과를 직접 보고서 인쇄물로 수거하거나 스프레드시트에 통계로 누적 관리할 수 있도록 백엔드 API 시스템을 도입했습니다.

### ① 실시간 API 데이터 통신 설계
* **POST (학생 ➡️ Sheets):** 
  * 학생이 엔딩 화면에서 '선생님께 제출하기' 버튼을 누르면, 모든 플레이 결과 및 서술형 에세이 내용이 `fetch` API를 통해 구글 배포 서버로 전송됩니다.
  * 이때, 대시보드 복원을 위해 **타임라인 전체 이력(`decisionsJson`)**과 **5개년 변화 수치(`historyJson`)**도 JSON 문자열 상태로 구글 시트의 17, 18번째 열에 완벽히 함께 압축 보관됩니다.
* **GET (Sheets ➡️ 교사용 대시보드):** 
  * 교사가 대시보드에 접근하면 저장된 전체 학생들의 데이터를 실시간으로 가져와 리스트 현황판을 렌더링합니다.

### ② 교사 대시보드 화면 및 학생 뷰 복원 시스템 구축
* **관리자 로그인 스크린 (`screen-admin-login`) 신설:**
  * 로그인 화면 우측 하단 링크를 통해 진입할 수 있으며, 기존 투박한 프롬프트 창 대신 깔끔하고 세련된 로그인 폼을 구현했습니다.
  * **인증 계정:** ID: `admin` / PW: `admin1204`
  * 키보드 편의 기능 탑재 (ID 입력 후 Enter ➡️ 비밀번호 필드로 자동 이동, 비밀번호 입력 후 Enter ➡️ 로그인 즉시 실행).
* **학생 뷰 복원 및 인쇄:** 
  * 대시보드 목록에서 특정 학생의 **[조회]** 버튼을 누르면, `gameState`에 해당 학생의 데이터가 임시 로드되어 **엔딩 스크린(`screen-ending`)으로 연동**됩니다.
  * 교사는 학생이 작성했던 에세이와 그래프, 결재 도장 역사를 그대로 화면에 복원하여 확인하고 즉시 인쇄(Ctrl+P)할 수 있습니다.
  * 조회 완료 시 다시 대시보드로 돌아갈 수 있는 **[대시보드로 돌아가기]** 버튼을 활성화하여 동선을 차별화했습니다.
* **학생 기록 개별 삭제 기능 탑재:**
  * 현황판 목록에 **빨간색 [삭제 🗑️]** 버튼을 신설했습니다.
  * 삭제를 승인하면 구글 스크립트 백엔드로 `action: "delete"` 포스트 요청을 날려, 해당 학생 고유 타임스탬프(`timestamp`)와 일치하는 구글 스프레드시트 내의 데이터를 완전히 제거한 후 현황판을 리로드합니다.
* **제출 성공 시 초기 화면 복귀 동선 추가:**
  * 최종 보고서 화면 최하단에서 '선생님께 제출하기' 버튼을 눌러 제출을 완수하면, 제출 버튼 옆에 **[처음 화면으로 가기 🏠]** 버튼이 동적으로 표출되도록 구현했습니다.
  * 이 버튼을 누르면 다음 학생이 곧바로 플레이할 수 있도록 로그인 폼(시장명, 도시명) 및 에세이 응답 창을 완전히 초기화하고 `screen-login` 화면으로 매끄럽게 되돌아갑니다.

## 6. CORS 프리플라이트 우회 및 디버깅 역사

구글 클라우드 보안 환경과 웹 브라우저의 보안 명세가 결합되어 발생하는 문제를 극복하기 위해 다년간의 백엔드 실무 노하우가 담긴 최적화 기법을 적용했습니다.

### ① CORS Preflight(OPTIONS 요청) 우회 (app.js)
* **문제점:** fetch 요청 시 `Content-Type: application/json` 등으로 요청을 보낼 경우 브라우저 수준에서 Preflight(OPTIONS) 통신을 구글 웹앱에 먼저 시도하게 되며, 이에 응답하지 못하면 CORS 오류를 뿜으며 데이터 전송 자체가 차단되었습니다.
* **해결책:** 
  1. 데이터를 전송할 때 `Content-Type`을 브라우저가 사전 심사를 거치지 않고 프리패스하는 단순 요청 규격인 **`text/plain`** 포맷으로 변경했습니다.
  2. 통신 모드를 **`no-cors`**로 강제 지정하여 구글의 리디렉션 주소(`script.googleusercontent.com`) 추적 과정에서 브라우저가 CORS 헤더 부재로 전송을 거부하는 현상을 완전히 차단했습니다. 데이터 적재 성공률 100%를 달성했습니다.

### ② 구글 스크립트 실행 오류 제거 (Code.gs)
* **문제점:** 초기 Apps Script 반환 객체에 존재하지 않는 `.setHeader('Access-Control-Allow-Origin', '*')` 메소드를 호출하여 `TypeError`가 발생했고, 구글 서버 내부 오류(500) 페이지가 출력되면서 CORS 차단으로 리다이렉션이 발생했던 현상입니다.
* **해결책:** 구글 Apps Script의 `ContentService.MimeType.JSON` 리턴 객체는 태생적으로 CORS 응답 헤더를 알아서 보장하므로, **해당 메소드를 모두 소스코드에서 제거**하여 연동 안정성을 극대화했습니다.

### ③ 백그라운드 Null 인스턴스 에러 극복
* **문제점:** 외부 웹앱이 스크립트를 비동기 백그라운드로 작동시킬 때 `SpreadsheetApp.getActiveSpreadsheet()`가 현재 인스턴스를 정상 인식하지 못해 데이터 기록 오류를 뿜는 현상입니다.
* **해결책:** 
  1. 구글 드라이브 단독 배포 시에도 스프레드시트를 강제로 찾아 매핑하도록 스크립트에 물리적인 **스프레드시트 ID (`SPREADSHEET_ID`) 명시적 대입 기능**을 신설했습니다.
  2. 활성 탭 혼선을 방지하고자 활성 시트 대신 물리적으로 항상 첫 번째 시트를 참조하는 **`ss.getSheets()[0]`** 연산자로 변경해 데이터 소실율을 Zero로 낮췄습니다.

### ③ 구글 스크립트 수정 시 주의사항
Apps Script 내부 코드를 조금이라도 수정하신 경우에는, 반드시 구글 Apps Script 편집창에서 **[배포] ➡️ [배포 관리] ➡️ [편집(연필)] ➡️ [버전: 새 버전] ➡️ [배포]**를 눌러 새 주소를 발급하고 이를 `app.js`에 대입해야만 변경 사항이 실시간 웹 환경에 갱신됩니다.
